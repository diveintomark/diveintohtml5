Offline Web Applications
Application cache is separate from browser cache; browsers may have separate limits for each.
HTTP semantics still apply to resources in cache manifest. Check your Expires and Cache-Control headers!
Test for support window.applicationCache
<html manifest> Links to cache manifest. Attribute must be in page markup; adding it later has no effect.
Events First four events are the most common, and they occur in order. 
checking      // always first event in sequence
downloading   // found manifest, fetching resources
progress      // still fetching resources
cached        // all resources cached, offline is a go!
noupdate      // manifest hasn't changed
updateready   // resources have changed, call swapCache() to activate
obsolete      // manifest has gone 404 (or 410)
error         // it all went wrong somewhere
Manifest syntax Must be served as text/cache-manifest. Check your Content-Type headers!
CACHE: Required resources. URLs can be absolute or relative to manifest. No wildcards.
If any required resource fails to load, application will not work offline.
NETWORK: Resources that are never cached. Accepts "*" wildcards.
FALLBACK: Pairs of resources. Second is used when first is unavailable. Matches partial URLs.
